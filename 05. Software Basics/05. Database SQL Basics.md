## 🧩 Task 2: Understanding Tables, Rows, and Columns

### 🧠 What is a Database?

A database is an organized collection of information stored on a computer.

Think of it as a digital notebook that:

- Never runs out of pages
- Stores large amounts of data
- Finds information quickly
- Allows sorting and filtering

### 📋 Information is Stored in Tables

Databases organize information into **tables**.

A table looks similar to a spreadsheet.

### 📊 Columns

Columns describe the type of information being stored.

Examples:

| Column |
|----------|
| Order ID |
| Drink |
| Price |
| Time |

Each column contains one type of data.

### 📄 Rows

Rows represent individual records.

Each row contains all information about one item.

For a café:

| Order ID | Drink | Price | Time |
|-----------|---------|---------|---------|
| 1 | Coffee | 3.50 | 09:15 |

This entire row represents one customer order.

### ☕ Café Example

Suppose a café records:

- Drink name
- Price
- Time purchased

Every customer order creates a new row.

For example:

| Order ID | Drink | Price |
|-----------|---------|---------|
| 1 | Coffee | 3.50 |
| 2 | Tea | 2.00 |
| 3 | Latte | 4.50 |

Three orders = Three rows.

### ➕ Adding Records

When a new customer places an order:

```text
New Order → New Row
```

### ➖ Removing Records

If an order is deleted:

```text
Only that row is removed
```

The rest of the table remains unchanged.

### 💬 Asking Questions with SQL

#### 🧠 What is SQL?

SQL stands for:

```text
Structured Query Language
```

SQL is used to communicate with databases.

Instead of manually searching through records, we ask the database questions.

### 🔍 Examples of Database Questions

A café owner might ask:

- Show all orders
- Show only coffee orders
- Show the cheapest drink
- Show the most expensive drink

These requests are called **queries**.

### 🧠 What is a Query?

A query is a request for information.

Queries:

- Retrieve data
- Filter data
- Sort data

Queries do **not** automatically change stored information.

They simply display results.

## 🧩 Task 3: Writing Your First SQL Query

### 🧠 Available Tables

The café database contains two tables:

### Orders

| Column |
|----------|
| id |
| drink |
| price |
| time |

### Menu

| Column |
|----------|
| drink |
| price |

### 🎯 Core SQL Keywords

Throughout this task you will use:

- `SELECT`
- `FROM`
- `WHERE`
- `ORDER BY`

These form the foundation of SQL querying.

### 📋 Step 1: View Everything (SELECT + FROM)

#### 🧠 SELECT *

The asterisk (`*`) means:

```text
All Columns
```

### 💻 Query

```sql
SELECT * FROM Orders;
```

### 🔍 What Happens?

The database:

1. Opens the Orders table
2. Retrieves every column
3. Displays every row

### 🧠 Breakdown

#### SELECT

Specifies what data to display.

#### *

Means all columns.

#### FROM

Specifies which table to use.

#### Orders

The table being queried.

### 📋 Step 2: Select Specific Columns

Sometimes we only need certain information.

### 💻 Query

```sql
SELECT drink, price FROM Orders;
```

### 🔍 Result

Only these columns appear:

- drink
- price

Other columns remain hidden.

### 🧠 Why Use Specific Columns?

Benefits:

- Cleaner output
- Faster queries
- Easier analysis

### 📋 Step 3: Filter Results (WHERE)

#### 🧠 Why Filter?

Sometimes we only want records matching specific conditions.

The `WHERE` keyword performs filtering.

### 💻 Query

```sql
SELECT * FROM Orders
WHERE drink = 'Coffee';
```

### 🔍 Result

Only rows where:

```text
drink = Coffee
```

are displayed.

### 🧠 How WHERE Works

Database checks every row:

```text
Is drink equal to Coffee?
```

If:

```text
Yes → Display row
No → Ignore row
```

### 💡 Helpful Query

To view available drinks:

```sql
SELECT * FROM Menu;
```

### 📋 Step 4: Sort Results (ORDER BY)

#### 🧠 Why Sort?

Sorting helps organize information.

Examples:

- Cheapest to most expensive
- Most expensive to cheapest
- Earliest to latest

### 💻 Query (Ascending)

```sql
SELECT * FROM Orders
ORDER BY price;
```

### 🔍 Result

Prices appear:

```text
Lowest → Highest
```

Ascending order is the default.

### 💻 Query (Descending)

```sql
SELECT * FROM Orders
ORDER BY price DESC;
```

### 🔍 Result

Prices appear:

```text
Highest → Lowest
```

### 🧠 DESC

`DESC` means:

```text
Descending Order
```

### 📋 Step 5: Combine Filtering and Sorting

Real SQL queries often combine multiple commands.

### 💻 Query

```sql
SELECT * FROM Orders
WHERE drink = 'Coffee'
ORDER BY price DESC;
```

### 🔍 What Happens?

Step 1:

```text
Find Coffee orders
```

Step 2:

```text
Sort Coffee orders
Highest Price → Lowest Price
```

Step 3:

```text
Display results
```

### 🧠 Key Insight

SQL commands can be combined together to answer more complex questions.
