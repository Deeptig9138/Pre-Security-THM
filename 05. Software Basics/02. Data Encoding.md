# 🔤 Data Encoding 

## 📘 Overview

The **Data Encoding** module explains how computers convert raw binary data into meaningful text, symbols, and emojis.

You already know that everything in a computer is stored as bits (0s and 1s). This module builds on that idea by showing how those bits become **characters we read and write every day**.

> 💡 **Core Idea:**  
> Encoding is the agreed system that maps numbers (bits) to meaningful characters like letters, symbols, and emojis.

- Understand Unicode as a universal standard

## 🧩 Task 1: Introduction

In computing, everything is stored as numbers. But how do numbers become letters, punctuation, or emojis?

That transformation happens through **encoding**.

### ⚠️ Problem Scenario

Sometimes when you open a file or webpage, you see unreadable symbols like:
```

```

This happens when:

- The file is saved using one encoding  
- But opened using a different encoding  

### 🧠 Key Concepts

- **Representation** → Data exists as bits in memory  
- **Encoding** → Agreement on what those bits mean  

For example:
- `01000001` → could mean the letter `A` (if using ASCII)

## 🔡 Task 2: ASCII

### 🧠 What is ASCII?

ASCII (American Standard Code for Information Interchange) is one of the earliest character encoding standards.

It uses **7 bits (0–127)** to represent:

- English letters  
- Numbers  
- Punctuation  
- Control characters  

### 📊 ASCII Concept

Each character is mapped to a number:

- `A` → 65  
- `B` → 66  
- `a` → 97  
- `0` → 48  

So computers store text like:
```
T → 01010100
r → 01110010
```

### 📄 Example: “TryHackMe”

Binary representation:
```
01010100 01110010 01111001 01001000 01100001 01100011 01101011 01001101 01100101 00001010
```

Hexadecimal representation:
```
54 72 79 48 61 63 6b 4d 65 0a
```

### 🌍 Limitations of ASCII

- Only supports English characters  
- Cannot represent accented or non-English letters  
- Only 128 characters total  

### 📌 European Extensions

To support more languages, standards like:

- ISO-8859-1 (Western Europe)  
- ISO-8859-2 (Central/Eastern Europe)  

were introduced, but compatibility issues remained.

## 🌐 Task 3: Unicode

### 🧠 Why Unicode Exists

ASCII and extended encodings were not enough for:

- Arabic scripts  
- Chinese and Japanese characters  
- Emojis  
- Mixed-language text  

So Unicode was created.

### 🌍 What is Unicode?

Unicode is a universal standard that assigns a unique code point to every character.

Examples:

- `A` → U+0041  
- `Ω` → U+03A9  
- `あ` → U+3042  

### 📊 Unicode Scale

Unicode supports:

- ~157,000 characters  
- Thousands of emojis  
- All modern writing systems  

#### **🧠 Key Idea**

> Unicode removes the need for multiple incompatible encoding systems.

### ⚙️ UTF-8, UTF-16, UTF-32

#### 🔹 UTF-8

- 1 to 4 bytes  
- Most widely used  
- Backward compatible with ASCII  

Examples:
- `A` → 1 byte  
- `🔥` → 4 bytes  

#### 🔹 UTF-16

- 2 or 4 bytes  
- Common for Windows systems  
- Some characters require surrogate pairs  

Example:
- `A` → U+0041  
- `🔥` → U+D83D U+DD25  

#### 🔹 UTF-32

- Fixed 4 bytes per character  
- Simple but memory heavy  

Example:
- `A` → U+00000041  
- `🔥` → U+0001F525  

#### **🧠 Key Insight**

> UTF-8 is efficient, UTF-32 is simple, UTF-16 is a middle ground.

## 🧠 Task 4: Conclusion

Encoding is the bridge between raw binary data and human-readable text.

---

## 🔑 Key Terminology

- **Encoding** → Mapping numbers to characters  
- **ASCII** → Early English-only encoding (7-bit)  
- **Unicode** → Universal character set  
- **UTF-8** → Variable-length encoding (1–4 bytes)  
- **UTF-16** → 2–4 byte encoding  
- **UTF-32** → Fixed 4-byte encoding  
- **Mojibake** → Garbled text due to encoding mismatch
