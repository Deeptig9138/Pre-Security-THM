# **How Websites Work**

This room explains **how websites are built and delivered** to users and introduces **fundamental security concepts** that every cybersecurity learner must understand before touching web vulnerabilities.

Understanding how websites work is critical because **most web attacks exploit misunderstandings between client-side and server-side behavior**.

---

## **Task 1: How Websites Work**

When you visit a website, your **browser (client)** sends a request to a **web server** asking for information about a specific page.

The server responds with data that your browser uses to render the website visually.

> A web server is simply a dedicated computer somewhere in the world that listens for requests and responds with data.

![web](https://github.com/Deeptig9138/Pre-Security-THM/blob/main/screenshots/web.png)

### Core Components of a Website

A website is made up of two main parts:

### **Front End (Client-Side)**
- Runs inside the user's browser
- Responsible for what the user sees and interacts with
- Built using:
  - HTML
  - CSS
  - JavaScript

### **Back End (Server-Side)**
- Runs on the server
- Handles requests
- Processes logic
- Communicates with databases
- Sends responses back to the client

> âš ï¸ **Security Insight:**  
> Anything running on the client-side can be seen, modified, and abused by attackers.

**Q)** What term best describes the component of a web application rendered by your browser?

## **Task 3: JavaScript**
**What is JavaScript?**

**JavaScript (JS) allows websites to become interactive and dynamic.**

Without JavaScript:

- Websites are static
- No real-time updates
- No interaction

With JavaScript:

- Buttons react to clicks
- Content updates dynamically
- Animations and logic execute in real time

**JavaScript in Action**

JavaScript can modify HTML elements dynamically:
```
document.getElementById("demo").innerHTML = "Hack the Planet";
```

HTML elements can also trigger JavaScript using events:

```
  Click Me!

```

## **Task 4: Sensitive Data Exposure**
**What is Sensitive Data Exposure?**

Sensitive Data Exposure occurs when a website leaks sensitive information in its frontend code.

Common examples include:

- Hardcoded passwords
- API keys
- Admin URLs
- Test credentials
- Hidden comments

Because HTML and JavaScript are visible to users, attackers often inspect the source code first.

**Why This Is Dangerous**

Attackers can:

- Discover credentials
- Access restricted areas
- Escalate privileges
- Move deeper into the application

> ðŸ§  Attacker Mindset:
> Always check the page source before attacking anything else.

## **Task 5: HTML Injection**
**What is HTML Injection?**

HTML Injection happens when:

- User input is not sanitised
- Input is directly rendered on the page
- The browser interprets it as HTML

This allows attackers to:

- Inject malicious content
- Modify page structure
- Insert external links
- Potentially escalate to XSS

**Why This Happens**

Developers:

- Trust user input
- Fail to strip HTML tags
- Directly inject input into the DOM

> ðŸ” Golden Rule:
> Never trust user input.

### Practical Exploitation

Injected a malicious HTML link:
```
<a href="http://hacker.com">Click Me</a>
```
