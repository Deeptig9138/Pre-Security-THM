# 🌐 Client-Server Basics 

## 📘 Overview

The **Client-Server Basics** module explains how computers communicate and provide services over a network.

In the early days, computers worked independently. Over time, systems became interconnected, leading to the development of networks like:

- ARPANET  
- CYCLADES  
- NPL  
- NSFNET  

These networks laid the foundation for the modern **internet**, where systems specialize and offer services to each other.

## 🧩 Task 1: Introduction

Modern computing relies on **interconnected systems** rather than isolated machines.

Just like people offer services based on skills, computers:

- Specialize in tasks  
- Provide services to other systems  
- Communicate over networks  

> 🧠 **Key Insight:**  
> The internet works because systems **request and provide services** to each other.

## 🍕 Task 2: Pizza Delivery Analogy

A simple way to understand client-server communication is through a **pizza ordering system**.

![Client-server analogy](https://tryhackme-images.s3.amazonaws.com/user-uploads/66c44fd9733427ea1181ad58/room-content/66c44fd9733427ea1181ad58-1768925163790.png)

### 🧑‍💻 Client & Server

- **Client** → Requests a service  
- **Server** → Provides the service  

#### Example

- Browser → Client  
- Website → Server  

> The client **always initiates** the request.

### 🔁 Request & Response

- Client sends a request  
- Server processes it  
- Server returns a response  

#### Possible Outcomes

- ✅ Success (resource delivered)  
- ❌ Error (invalid request or unavailable resource)  

### 📡 Protocol

A **protocol** defines how communication happens.

It specifies:

- Commands (e.g., GET)  
- Structure of requests  
- Syntax/language  
- Expected responses  
- Error handling  

> 🧠 Think of it as the **language** computers use.

### 🚪 Port

A **port** identifies a specific service on a server.

#### Analogy

- One building (server)  
- Multiple doors (ports)  
- Each door → different service  

### 🌍 DNS (Domain Name System)

DNS converts human-readable names into IP addresses.

#### Example

- `google.com` → `142.250.x.x`

> 🧠 Like a GPS translating names into coordinates.

### ✅ Answers

**Q)** What identifies a specific service on a server?  
**A)** `port`

**Q)** What is the address of a server called?  
**A)** `Internet Protocol Address`

## 🌐 Task 3: Web Communication in Practice

### 📡 HTTP/HTTPS

- **HTTP (Hypertext Transfer Protocol)**  
- **HTTPS (Secure version)**  

#### Key Property

- Stateless → Each request is independent  

### 🔐 State Management

Since HTTP is stateless:

- Websites use **sessions, cookies, tokens**  
- These maintain user state (e.g., login sessions)  

### 📜 HTTP Methods

There are 9 core methods:

- GET  
- POST  
- PUT  
- DELETE  
- PATCH  
- HEAD  
- OPTIONS  
- CONNECT  
- TRACE  

### 📥 GET Method

Used to **retrieve data** from a server.

#### Example
`GET https://tryhackme.com/index.php`

- Browser sends request  
- Server responds with:
  - Status code  
  - Requested content  


### 🔍 Request Breakdown

When inspecting a request:

- **Scheme** → Protocol (HTTP/HTTPS)  
- **Host** → Domain name  
- **Filename** → Resource requested  
- **Address** → IP address  
- **Status** → Result (e.g., 200 OK)  

### 📦 Response Structure

- **Header** → Metadata  
- **Body** → Actual content (HTML, data, etc.)  

### ✅ Answers

**Q)** Host in `https://www.iamlearning.thm/contact`  
**A)** `www.iamlearning.thm`

**Q)** Scheme in the same URL  
**A)** `https`
