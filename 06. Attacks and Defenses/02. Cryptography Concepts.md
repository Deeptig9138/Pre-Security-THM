# 🔐 Cryptography Concepts

## 📘 Overview

The **Cryptography Concepts** module introduces one of the most important areas of cybersecurity: protecting information through encryption.

Every day, sensitive information such as passwords, banking details, medical records, and private messages travels across the internet. Cryptography helps ensure that this information remains private and trustworthy, even if attackers can see the data as it moves between systems.

> 💡 **Core Idea:**  
> Cryptography protects information by converting readable data into unreadable data that only authorized people can restore and understand.

## 🧩 Task 2: Hiding Information – Symmetric Encryption

### ❓ Opening Question

If someone can monitor every piece of data travelling between two people, how can those people still exchange secrets?

### 🧠 Core Cryptography Terms

Before learning encryption, we need four important concepts.

### 📄 Plaintext

Plaintext is information that can be read normally.

Examples:

```text
HELLO
```

```text
Patient Name: Alice Smith
```

### 🔒 Ciphertext

Ciphertext is encrypted information that should not make sense to someone who lacks the key.

Examples:

```text
KHOOR
```

```text
Sdwlhqw Qdph: Dolfh Vplwk
```

### 🔑 Key

A key is a secret value used during encryption and decryption.

Think of it like a password used by the encryption process.

### ⚙️ Algorithm

An algorithm is the public set of instructions used to transform data.

Important:

> Security comes from protecting the key, not hiding the algorithm.

### 🔄 Encryption and Decryption

#### Encryption

```text
Plaintext + Algorithm + Key
                ↓
           Ciphertext
```

#### Decryption

```text
Ciphertext + Algorithm + Key
                 ↓
            Plaintext
```

### 📦 The Lockbox Analogy

Imagine a lockbox.

#### Algorithm

How the lock works.

Everyone can understand the lock mechanism.

#### Key

The physical key.

Only authorized people possess it.

#### Plaintext

The letter inside the lockbox.

#### Ciphertext

The locked box moving through the postal system.

### Example

Alice wants to send Bob a secret message.

She:

1. Writes a message
2. Places it inside a lockbox
3. Locks the box
4. Sends it through the mail

Bob:

1. Receives the lockbox
2. Uses the same key
3. Opens it
4. Reads the message

This is **symmetric encryption**.

The same key:

- Locks the message
- Unlocks the message

### 📄 Plaintext vs Ciphertext

Suppose Alice writes:

```text
HELLO
```

This is plaintext.

After encryption:

```text
KHOOR
```

This is ciphertext.

Anyone without the key sees meaningless text.

Bob uses the same key to restore:

```text
HELLO
```

### 🏛️ The Caesar Cipher

To understand encryption, we use the Caesar Cipher.

⚠️ This cipher is not secure today.

It is only used for learning purposes.

### How It Works

Every letter shifts by a fixed amount.

The shift amount is the key.

Example:

#### Key = 3

```text
A → D
B → E
C → F
```

and

```text
X → A
Y → B
Z → C
```

#### Encrypting HELLO

Using key 3:

```text
H → K
E → H
L → O
L → O
O → R
```

Result:

```text
KHOOR
```

#### Decrypting KHOOR

Shift backwards by 3:

```text
K → H
H → E
O → L
O → L
R → O
```

Result:

```text
HELLO
```

### 🧠 Key Insight

In the Caesar Cipher:

#### Algorithm

```text
Shift letters by a number
```

Public knowledge.

#### Key

```text
3
```

Secret information.

### 🔐 Symmetric Encryption Explained

The Caesar Cipher is a form of symmetric encryption.

Characteristics:

- Same key encrypts data
- Same key decrypts data
- Both users need the same secret key

### Advantages

#### ⚡ Fast

Encrypts large amounts of data quickly.

#### 💾 Efficient

Ideal for:

- Files
- Hard drives
- Network traffic

### The Problem: Key Distribution

How do Alice and Bob safely exchange the key?

If attackers intercept the key:

- They can read all future messages.

This challenge is called:

> **Key Distribution Problem**

## 🧩 Task 3: Sharing Keys Safely – Asymmetric Encryption

### ❓ Opening Question

If Alice and Bob have never met before, how can they exchange encrypted messages safely?

### 🚨 The Key Distribution Problem

Symmetric encryption requires a shared secret key.

But how do two strangers share that key securely?

Asymmetric encryption solves this problem.

### 🔑 Two Keys Instead of One

Asymmetric encryption uses:

#### Public Key

Can be shared with anyone.

#### Private Key

Must remain secret.

#### Rule

If data is encrypted using:

```text
Public Key
```

it can only be decrypted using:

```text
Private Key
```

### 📬 Mailbox Analogy

Imagine a public mailbox.

#### Public Key

The mail slot.

Anyone can insert mail.

#### Private Key

The mailbox door key.

Only the owner can open it.

### Example

Alice wants to send Bob a message.

1. Bob publishes his public key
2. Alice encrypts using Bob's public key
3. Bob decrypts using his private key

Even if attackers intercept the message, they cannot read it.

### ✅ Solving the Key Distribution Problem

Steps:

1. Bob creates public and private keys
2. Bob shares his public key
3. Alice encrypts data using the public key
4. Bob decrypts using the private key

No secret key needs to be shared beforehand.

### 🌐 Real-World Example: HTTPS

Every time you visit:

```text
https://google.com
```

or

```text
https://tryhackme.com
```

asymmetric encryption helps secure the connection.

### HTTPS Process

#### Step 1

Browser requests website public key.

#### Step 2

Website sends a certificate.

#### Step 3

Browser and website securely establish a shared symmetric key.

#### Step 4

Both sides switch to symmetric encryption.

### 🤝 Hybrid Encryption

Modern systems use both encryption types together.

#### Asymmetric Encryption

Used for:

- Identity verification
- Secure key exchange

#### Symmetric Encryption

Used for:

- Actual data transfer

because it is much faster.

### 📜 Certificates

How does your browser know a public key really belongs to the correct website?

Through certificates.

#### A Certificate Contains

- Website identity
- Public key
- Digital signature from a trusted authority

#### Certificate Authorities (CAs)

Trusted organizations that verify identities.

Browsers trust certificates signed by trusted CAs.

#### Browser Verification

The browser checks:

- Certificate signature
- Certificate validity dates
- Certificate trust chain

If valid:

🔒 Padlock appears.

If invalid:

⚠️ Browser warning appears.

#### 🔍 Viewing Certificates

You can inspect certificates yourself.

Steps:

1. Visit an HTTPS website
2. Click the padlock icon
3. Open certificate details
4. View:

- Issued To
- Issued By
- Valid From
- Valid Until

### ⚖️ Symmetric vs Asymmetric Encryption

| Feature | Symmetric Encryption | Asymmetric Encryption |
|----------|----------|----------|
| Keys | One key | Public + Private key |
| Key Sharing | Difficult | Easy |
| Speed | Fast | Slower |
| Usage | Bulk data encryption | Key exchange and certificates |
| Analogy | One key opens and closes lockbox | Mailbox with public slot and private door |
