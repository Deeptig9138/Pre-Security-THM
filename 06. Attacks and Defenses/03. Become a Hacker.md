# 🌐 Become a Hacker 

## 📘 Overview

**Offensive Security** focuses on proactively testing systems by attempting to break into them to identify weaknesses before real attackers can exploit them.

Unlike defensive security, which focuses on protection and monitoring, offensive security takes an attacker’s perspective to discover vulnerabilities and assess risk.

In this module, you'll learn how ethical hackers think, discover weaknesses, and safely test systems within an authorized scope. You'll also complete a hands-on exercise involving web application assessment, enumeration, and password attacks.

> 💡 **Core Idea:**  
> Ethical hackers use the same techniques as attackers, but with permission and a goal of improving security.

## 🧩 Task 1: What Is Offensive Security?

Cyber security is often divided into two broad areas:

- Defensive Security (protecting systems)
- Offensive Security (testing systems)

Offensive security professionals proactively attempt to identify weaknesses before malicious attackers can exploit them.

Rather than asking:

> "How do we defend this system?"

They ask:

> "How could someone break into this system?"

### 🧠 Thinking Like an Attacker

Hackers constantly ask questions such as:

- What is exposed?
- What can be accessed?
- What assumptions does the system make?
- What happens if I use unexpected input?

This process helps uncover weaknesses that developers or administrators may have overlooked.

### ⚖️ Ethical Hacking

In this room, hacking refers to:

- Authorized testing
- Legal testing
- Controlled testing
- Security improvement

This process is commonly called:

- Penetration Testing
- Ethical Hacking

The purpose is to discover weaknesses so they can be fixed before real attackers find them.

## 🧩 Task 2: Finding Weaknesses

Before performing an assessment, it's important to understand common offensive security terminology.

### 📚 Core Offensive Security Terms

#### 🔴 Red Teaming

A structured and authorized attack simulation that mimics real-world adversaries.

Purpose:

- Test defenses
- Measure detection capabilities
- Identify weaknesses

#### 🎯 Penetration Testing

A controlled security assessment where testers attempt to exploit vulnerabilities within an approved scope.

Purpose:

- Understand risk
- Validate vulnerabilities
- Improve security

#### ⚠️ Vulnerability

A weakness in:

- Software
- Hardware
- Configuration
- Processes

that could be abused by attackers.

#### 💥 Exploit

A technique used to take advantage of a vulnerability.

Examples:

- Accessing restricted pages
- Gaining unauthorized access
- Executing unintended actions

#### 📏 Scope

The boundaries of an engagement.

Scope defines:

- What can be tested
- What cannot be tested
- Allowed actions
- Restricted actions

### 🔑 The Most Important Rule

All ethical hacking requires:

> Permission

Without permission, testing systems is illegal and unethical.

### 🏢 Real-World Scenario

Mike has created an online shop and wants a security assessment before launch.

Your goal:

- Identify hidden pages
- Discover weaknesses
- Help secure the application

Target:

```text
http://www.onlineshop.thm/
```

### 🔍 Manual Enumeration

One method of discovering hidden pages is testing common paths manually.

Examples:

```text
/sitemap
/mail
/register
/login
/admin
```

When a page doesn't exist:

```text
Error 404
```

appears.

### 🎯 Hidden Page Discovered

The hidden page found was:

```text
/login
```

### 🤖 Automated Enumeration

Testing a few pages manually is manageable.

Testing thousands is not.

This is where automated tools help.

### 🛠️ Gobuster

Gobuster is a directory enumeration tool.

Example:

```bash
gobuster dir --url http://www.onlineshop.thm/ -w /usr/share/wordlists/dirbuster/directory-list.txt
```

### 🔍 Command Breakdown

#### gobuster

The enumeration tool.

#### dir

Directory discovery mode.

#### --url

Specifies the target.

```bash
http://www.onlineshop.thm/
```

#### -w

Specifies the wordlist.

```bash
/usr/share/wordlists/dirbuster/directory-list.txt
```

### 🎯 Results

#### Hidden Page

```text
/login
```

#### Status Code

```text
200
```

Meaning:

```text
Page exists and is accessible.
```

## 🧩 Task 3: Exploiting Weaknesses

Finding a weakness is only the beginning.

Attackers often chain multiple weaknesses together.

### 🧩 The Domino Effect

A single weakness may not be dangerous.

However, combining weaknesses can create serious security risks.

Example:

1. Hidden login page
2. Weak password
3. Administrative access

Together these weaknesses create a much bigger issue.

### 🧠 Think Like a Hacker

Hackers don't ask:

> Does this work?

They ask:

> How can this be abused?

### Key Principles

#### Ask Questions

```text
What if it doesn't work correctly?
```

#### Test the Unexpected

Try inputs developers may not have anticipated.

#### Chain Weaknesses

Combine small flaws to create greater impact.

#### Think Like an Adversary

Ask:

```text
How would an attacker approach this?
```

### 🎯 Why Credentials Matter

Attackers often target:

- Usernames
- Passwords
- Authentication systems

because access unlocks valuable resources.

### 🔓 What Can Authenticated Access Reveal?

#### Sensitive Functionality

Actions that should only be available to authorized users.

#### User Data

Examples:

- Names
- Emails
- Account information

#### Administrative Features

Capabilities such as:

- Managing users
- Changing settings
- Controlling the application

#### Additional Attack Opportunities

Access often exposes new vulnerabilities.

### 🤖 Hacking Automation

Testing five passwords manually is easy.

Testing thousands is not.

Automation makes the process faster.

### 🛠️ Hydra

Hydra is a password-testing tool.

It performs:

```text
Dictionary Attacks
```

using predefined wordlists.

### 💻 Hydra Command

```bash
hydra -l admin -P passlist.txt www.onlineshop.thm http-post-form "/login:username=^USER^&password=^PASS^:F=incorrect" -V
```

### 🔍 Command Breakdown

#### hydra

Password attack tool.

---

#### -l admin

Username:

```text
admin
```

#### -P passlist.txt

Password wordlist.

#### www.onlineshop.thm

Target website.

#### http-post-form

Specifies login form testing.

#### Login String

```text
/login:username=^USER^&password=^PASS^:F=incorrect
```

Defines:

- Login endpoint
- Parameters
- Failure message

#### -V

Verbose mode.

Shows every login attempt.

### 📖 What Is a Dictionary Attack?

A dictionary attack tries passwords from a predefined list.

Examples:

```text
password
123456
qwerty
admin123
```

rather than attempting every possible combination.
